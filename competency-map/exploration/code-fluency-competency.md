### Competency: Code Fluency

The ability to use programming languages to create solutions that are not only functional but also
clear, maintainable, and idiomatic, making them effective tools for collaboration and long-term value. 
This includes the effective consumption, design, and implementation of libraries, frameworks, and APIs.

| PL  | Definition & Scope                                                                                                                                                                            | Key Indicators & Assessable Behaviors                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
|:----|:----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|:---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| PL1 | Writes functional, procedural scripts that achieve a specified, immediate goal. Code is correct but may be inefficient or hard to read.                                                       | <ul><li>Library Usage: Can successfully install and use functions from standard libraries (`pandas`, `requests`, `scikit-learn`).</li><li>API Usage: Can read API documentation and write code to successfully call an existing API to retrieve or send data.</li><li>Code Quality: Scripts are functional for the author's immediate use but may lack comments, clear structure, or adherence to style guides.</li></ul>                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| PL2 | Writes clean, well-structured, and documented code that adheres to idiomatic patterns of the primary language and community best practices.                                                   | <ul><li>Idiomatic Code: Consistently uses language-specific idioms (e.g., comprehensions in Python instead of manual loops).</li><li>Framework Usage: Effectively uses the core APIs of major frameworks (e.g., `scikit-learn` Pipelines, `pandas` data manipulation).</li><li>Simple API Design: Can design and implement a simple, consistent RESTful API for their models or analyses, typically using a framework like FastAPI. The API is reliable and easy for others to consume.</li></ul>                                                                                                                                                                                                                                                                                                                                                                                                  |
| PL3 | Designs and builds robust, maintainable, and high-leverage assets. Exhibits "taste" in selecting tools, programming patterns, or language features to architect elegant and robust solutions. | <ul><li>Library/Framework Creation: Can design and build a reusable library (like `fastapi`) or package for use by other team members, following best practices for structure, testing, and documentation.</li><li>Advanced API Design: Can design and implement complex APIs, correctly applying advanced patterns such as asynchronous programming for performance, or adhering to rigorous design standards (like Google AIPs) for consistency at scale.</li><li>Taste & Judgment: Can articulate the trade-offs between different libraries or frameworks and justify their choices based on "Pythonic" (or "R-like") design philosophy, maintainability, and fitness for a specific task.</li><li>Advanced Design: Designs and builds solutions using advanced patterns like custom decorators, context managers, or asynchronous programming. Exhibits "taste" in selecting tools.</li></ul> |
| PL4 | Conceives and designs fundamental, high-leverage tools, frameworks, or standards that are adopted across the organization or even by the public community.                                    | <ul><li>Fundamental Creation and Standardization: Invents or defines new standards for how systems interact (e.g., creating or contributing to data exchange formats like gRPC or Arrow).</li></ul>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
